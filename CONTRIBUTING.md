Open source projects should include a Contributing file that describes how someone in the community can contribute to the project. The Mozilla Science Lab has an article on [Wrangling Web Contributions: How to Build a CONTRIBUTING.md](https://mozillascience.github.io/working-open-workshop/contributing/)

# Contributing to Flow

We welcome contributions to Flow! If you would like to contribute, please fork the repository and submit a pull request. We will review your pull request and provide feedback. We will also add you as a contributor to the project if your pull request is accepted.

## Code of Conduct

We expect all contributors to follow our code of conduct. We want to create a
welcoming and inclusive community where everyone feels safe and respected.
Please read our [Code of Conduct](https://github.com/ritual-17/flow/blob/main/CodeOfConduct.md) before contributing to
the project.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on our GitHub repository. We will review your issue and provide feedback. We will also add you as a contributor to the project if your issue is accepted.
